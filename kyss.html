What is the structure of the files containing protein and peptide lists?
- Each protein list MUST have the name: WHATEVER_proteingroups.txt, where WHATEVER can be any word.
- Each peptide list MUST have the name: WHATEVER_psms.txt, where WHATEVER can be any word, but IT HAS TO BE THE SAME than its protein list counterpart.
- Both lists consist in text files (no format), saved in columns format separated by TABS. Each column has a header in the first lane, and this header HAS TO BE the ones listed below.
(these can or cannot be in double quotes) (the order is not important) (a column not present is usually ignored) (not all columns are used by KYSS):
- HEADERS for PROTEINS lists:
"Accession" "Description" "Score" "Coverage" "# Proteins" "# Unique Peptides" "# Peptides" "# PSMs" "Area" "# AAs" "MW [kDa]" "calc. pI"
- HEADERS for PEPTIDES lists:
"Confidence Level" "Search ID" "Processing Node No" "Sequence" "Unique Sequence ID" "PSM Ambiguity" "Protein Descriptions" "# Proteins" "# Protein Groups" "Protein Group Accessions" "Modifications" "Activation Type" "DeltaScore" "DeltaCn" "Rank" "Search Engine Rank" "Precursor Area" "QuanResultID" "q-Value" "PEP" "Decoy Peptides Matched" "Exp Value" "Homology Threshold" "Identity High" "Identity Middle" "IonScore" "Peptides Matched" "# Missed Cleavages" "Isolation Interference [%]" "Ion Inject Time [ms]" "Intensity" "Charge" "m/z [Da]" "MH+ [Da]" "Delta Mass [Da]" "Delta Mass [PPM]" "RT [min]" "First Scan" "Last Scan" "MS Order" "Ions Matched" "Matched Ions" "Total Ions" "Spectrum File" "Annotation"
Basic instructions:
1) Drag your protein/peptide pair of lists in the grey panel. You can drag multiple files at once or sequentially. The order of addition will be keep later on.
2) Once all your lists are dragged, clik the button to start the analysis
